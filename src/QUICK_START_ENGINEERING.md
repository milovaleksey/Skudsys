# ⚡ Быстрый старт: Инженерный раздел

## 🎯 Что нужно сделать

### Шаг 1: Выполнить миграцию БД ✅

```bash
# Из корневой директории проекта
node backend/add-engineering-permission.js
```

**Результат**: Роли `admin` и `security` получат доступ к инженерному разделу

---

### Шаг 2: Перезапустить backend сервер ✅

```bash
# Остановите текущий процесс (Ctrl+C)
# Затем запустите снова:

cd backend
npm run dev
```

**Проверка**: В логах должна появиться строка:
```
[MQTT] ✅ Подписка на топик аномальных событий: Skud/baddialsevent
```

---

### Шаг 3: Готово! 🎉

Откройте приложение и:
1. Войдите под пользователем с ролью `admin` или `security`
2. В боковом меню найдите **"Инженерный раздел"** (иконка 🔧)
3. Используйте функционал:
   - Просмотр аномальных событий
   - Фильтрация и поиск
   - Экспорт в Excel
   - Управление правилами доступа

---

## 🔧 Настройка MQTT (опционально)

Если нужны live-обновления аномальных событий:

1. Убедитесь что MQTT брокер работает
2. Публикуйте события в топик: `Skud/baddialsevent`
3. Формат: JSON массив с событиями

**Пример**:
```json
[
  {
    "time_label": "2026-03-12T10:30:00.000Z",
    "Тип_события": "Идентификатор не зарегистрирован",
    "ФИО_пользователя": "Иванов Иван Иванович",
    "UPN": "ivanov@utmn.ru",
    "Zone": "Корпус №1",
    "Child_Zone": "1 этаж",
    "Device": "К1 Турникет №1 устр",
    "identificator": 123456
  }
]
```

## ❓ Проблемы?

### Нет доступа к странице
→ Проверьте что миграция выполнена и сервер перезапущен

### MQTT не подключается
→ Проверьте `.env`: `MQTT_ENABLED=true` и настройки брокера

### События не отображаются
→ Проверьте публикацию в топик `Skud/baddialsevent`
