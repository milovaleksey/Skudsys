# ⚡ Быстрое исправление Vite

## Что случилось?

У вас на сервере работает **корневая структура**, но файлы во `/frontend/` пытались импортировать компоненты которых там нет.

## ✅ Что исправлено

1. **`/frontend/vite.config.ts`** - алиасы теперь указывают на корневые папки `../components`, `../contexts` и т.д.
2. **`/frontend/App.tsx`** - импорты обновлены на `../components/LoginPage` вместо `./components/LoginPage`
3. **`/frontend/main.tsx`** - путь к стилям обновлен на `../styles/globals.css`

## 🚀 Как запустить

### На сервере (production) - БЕЗ ИЗМЕНЕНИЙ
```bash
# Ваша рабочая команда остается той же
npm run dev          # или
npm run build
```

### Для разработки (локально)
```bash
# Запуск из корня (рекомендуется)
npm run dev

# ИЛИ запуск из frontend (теперь тоже работает)
cd frontend
npm run dev
```

## 🔍 Проверка

```bash
# Сделать скрипты исполняемыми
chmod +x check-structure.sh start-frontend.sh

# Проверить структуру
./check-structure.sh

# Автозапуск из правильной папки
./start-frontend.sh
```

## ❗ Важно

**На production сервере ничего менять не нужно!** Ваша корневая структура работает правильно.

Исправления нужны только если вы пытались запустить из `/frontend/` директории.

## 📝 Что делать дальше

1. Убедитесь что на сервере вы запускаете из **корня проекта**
2. Проверьте что nginx раздает `/dist` из корня, а не из `/frontend/dist`
3. Используйте корневую структуру - она стабильная и рабочая
