- Установлен правильный адрес MQTT брокера: `10.101.221.198`


## ⚠️ Если проблемы остались:

### Проблема: Backend всё ещё подключается к localhost:1883

**Решение:**
1. Убедитесь, что файл `/backend/.env` существует
2. Проверьте содержимое:
   ```bash
   cat /var/www/utmn-security/debug/backend/.env | grep MQTT
   ```
   Должно быть:
   ```
   MQTT_BROKER=10.101.221.198
   MQTT_PORT=1883
   ```
3. **ПЕРЕЗАПУСТИТЕ backend** (Ctrl+C → npm start)

---

### Проблема: 429 Too Many Requests

**Решение:**
1. Полностью остановите backend
2. Очистите кэш браузера (Ctrl+Shift+Delete)
3. Запустите backend заново
4. Обновите страницу в браузере (Ctrl+Shift+R)

---

### Проблема: WebSocket всё ещё не работает

**Проверьте в логах backend:**
```
[MQTT WebSocket] Инициализирован для /ws/mqtt
[Parking WS] ✅ WebSocket инициализирован для /parking-ws
```

Если этого нет - backend не запустился корректно.

**Проверьте ошибки:**
```bash
cd /var/www/utmn-security/debug/backend
npm start 2>&1 | tee backend.log
```

---

### Проблема: MQTT не подключается

**Если видите:**
```
[MQTT] ❌ Ошибка подключения: connect ECONNREFUSED 10.101.221.198:1883
```

**Это нормально!** Сервер продолжит работать.

**Проверьте:**
1. Доступен ли MQTT сервер:
   ```bash
   telnet 10.101.221.198 1883
   # Или
   nc -zv 10.101.221.198 1883
   ```

2. Если MQTT недоступен, это не критично:
   - REST API будет работать
   - WebSocket будет работать
   - Только MQTT данные не будут обновляться в реальном времени
