# 🔧 Скрипты для настройки аналитики

## ⚠️ Проблема: "Показывается только 10 зон/корпусов"

Если на странице "Аналитика" показывается только 10 зон/корпусов вместо всех, это означает что:

1. ❌ **Не отправлена конфигурация аналитики** в MQTT топик `Skud/analytics/config`
2. ❌ **Или лимит в конфигурации слишком маленький** (по умолчанию 10)
3. ❌ **Или данные не отправлены** в топик `Skud/analytics/events/aggregated`

## ✅ Решение

Используйте скрипты ниже для настройки MQTT топиков.

## Отправка данных аналитики

### Из CSV файла

Если у вас есть CSV файл с данными (как `/imports/rez.csv`):

```bash
cd backend
node scripts/publish-analytics-data.js ../imports/rez.csv
```

**Что делает скрипт:**
- Читает CSV файл
- Парсит данные (поддержка кавычек, запятых внутри значений)
- Публикует в топик `Skud/analytics/events/aggregated`
- Показывает статистику: количество зон, дат, событий

### Формат CSV

```csv
event_date,root_zone_name,total_events
2025-06-18,"Корпус №11",19
2025-06-18,"Корпус №16",3
2025-06-19,"Корпус № 4 Приемная комиссия",190
```

**Важно:**
- Первая строка - заголовки
- Названия зон могут быть в кавычках (особенно если содержат пробелы/запятые)
- total_events должно быть числом

## Полный процесс настройки

### 1. Отправить конфигурацию
```bash
cd backend
node scripts/publish-analytics-config.js
```

### 2. Отправить данные
```bash
node scripts/publish-analytics-data.js ../imports/rez.csv
```

### 3. Запустить/перезапустить backend
```bash
npm start
```

### 4. Проверить frontend
Откройте http://localhost:5173/analytics

## Проверка

После выполнения шагов:

✅ В консоли backend должно появиться:
```
[MQTT] 📊 Получена конфигурация аналитики: Топ активных зон (limit: 100)
```

✅ На странице "Аналитика" в фильтре "Корпус/Здание" должны отображаться все здания (~20-30 вместо 10)

✅ В "Детальном отчете по корпусам" должны быть все здания

## Дополнительно

### Изменить лимит топ зон

Откройте файл `scripts/publish-analytics-config.js` и измените:

```javascript
{
  id: "top_zones",
  config: {
    limit: 100, // ← Измените на нужное значение
    // ...
  }
}
```

Затем запустите скрипт снова.

### Отладка

Посмотреть логи backend:
```bash
# В консоли где запущен backend должны быть сообщения:
[MQTT] 📊 Получена конфигурация аналитики
[MQTT] 📊 Получено XXX записей данных аналитики
[MQTT] ✅ Обработано 10 типов аналитики
```

Посмотреть логи frontend (F12 → Console):
```
[Analytics WebSocket] ✅ Подключено
[Analytics WebSocket] Получены начальные данные аналитики
🔍 Нормализация названий зданий: {...}
```

## Структура данных

Данные аналитики отправляются в топик `Skud/analytics/events/aggregated` в формате:

```json
[
  {
    "event_date": "2025-06-18",
    "root_zone_name": "Корпус №11",
    "total_events": 19
  },
  {
    "event_date": "2025-06-18",
    "root_zone_name": "Корпус №16",
    "total_events": 3
  }
]
```

Backend обрабатывает эти данные согласно конфигурации и отправляет через WebSocket на frontend.
