# Инструкции по перезапуску backend

## Проблема
После изменений в `/backend/src/controllers/user.controller.js` бэкенд-сервер нужно перезапустить, чтобы изменения вступили в силу.

## Решение

### 1. Перезапуск через PM2 (если используется)
```bash
cd backend
pm2 restart security-backend
# или
pm2 restart all
```

### 2. Перезапуск через systemd (если используется)
```bash
sudo systemctl restart security-backend
```

### 3. Ручной перезапуск
```bash
# Найти процесс
ps aux | grep node

# Убить процесс (замените PID на реальный ID процесса)
kill -9 PID

# Запустить снова
cd backend
npm start
# или для продакшена
NODE_ENV=production npm start
```

## Что было изменено

### Backend (`/backend/src/controllers/user.controller.js`)
1. ✅ **Поддержка boolean и числовых значений для `isActive`**:
   - Схемы `createUserSchema` и `updateUserSchema` теперь принимают как `boolean`, так и числа `0` и `1`
   - Добавлено преобразование числовых значений в boolean после валидации

2. ✅ **Поддержка смены пароля через API**:
   - Добавлено поле `password` в схему валидации `updateUserSchema` с параметрами `.min(8).optional().allow('')`
   - Реализована обработка обновления пароля с хешированием через bcrypt
   - Добавлена запись в audit log при смене пароля (с маскировкой `*****`)

3. ✅ **Улучшенное логирование для отладки**:
   - Добавлено логирование входящих запросов на обновление пользователя
   - Добавлено логирование ошибок валидации с деталями

### Frontend
1. ✅ **Компонент `PasswordRequirements.tsx`**:
   - Визуальная валидация пароля в реальном времени
   - Отображение требований: минимум 8 символов, заглавные/строчные буквы, цифры
   - Индикаторы выполнения каждого требования (✓/✗)

2. ✅ **Обновлен `UsersSettingsPage.tsx`**:
   - Интегрирован компонент `PasswordRequirements` в формы создания и редактирования
   - Упрощена логика смены пароля - пароль просто добавляется в `updateData` если введен
   - Добавлены подсказки о минимальных требованиях к паролю

## Проверка работы
После перезапуска сервера:
1. Откройте консоль браузера (F12)
2. Попробуйте изменить пароль пользователя
3. ✅ В консоли браузера не должно быть ошибки 400
4. ✅ В логах сервера должна появиться запись: `🔧 Update user request: { id: X, body: {...} }`
5. ✅ Пароль успешно обновится и будет сохранен в базе данных

- ❌ Отсутствие подсказок о требованиях к паролю → ✅ Добавлен визуальный компонент с валидацией
