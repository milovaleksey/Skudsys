## 🎯 Примеры использования

### Пример 1: Все страны, базовый порог (3 дня)

**Вход:**
```sql
CALL sp_foreign_students_missing('all', 3);
```

**Обработка:**
```
persons (иностранные студенты)
    ↓
JOIN person_countries (кроме России)
    ↓
JOIN events (последний визит)
    ↓
DATEDIFF(NOW(), MAX(event_time)) >= 3
    ↓
ORDER BY days_missing DESC
```

**Выход:**
```
+-----+---------------------------+-------------+--------------+
| fio | country                   | last_seen   | days_missing |
+-----+---------------------------+-------------+--------------+
| А.А.| ЕГИПЕТ                    | 2026-03-02  | 7            |
| Н.Н.| КАЗАХСТАН                 | 2026-03-04  | 5            |
| Л.В.| КИТАЙ                     | 2026-03-05  | 4            |
+-----+---------------------------+-------------+--------------+
```

---

### Пример 2: Конкретная страна, критический порог (7 дней)

**Вход:**
```sql
CALL sp_foreign_students_missing('КАЗАХСТАН', 7);
```

**Обработка:**
```
persons (person_type = 'student')
    ↓
JOIN person_countries (country_name = 'КАЗАХСТАН')
    ↓
JOIN events (последний визит)
    ↓
DATEDIFF(NOW(), MAX(event_time)) >= 7
    ↓
ORDER BY days_missing DESC
```

**Выход:**
```
+-----+---------------------------+-------------+--------------+
| fio | country                   | last_seen   | days_missing |
+-----+---------------------------+-------------+--------------+
| Н.Н.| КАЗАХСТАН                 | 2026-03-01  | 8            |
+-----+---------------------------+-------------+--------------+
```

## 📐 Диаграмма времени выполнения

```
Запрос → [ 1-50ms ]  → SQL парсинг
       ↓
       → [ 50-200ms ] → Чтение таблиц (persons, events)
       ↓
       → [ 20-100ms ] → JOIN операции
       ↓
       → [ 10-50ms ]  → GROUP BY и агрегация
       ↓
       → [ 5-20ms ]   → HAVING фильтрация
       ↓
       → [ 5-10ms ]   → ORDER BY сортировка
       ↓
       → [ 1-5ms ]    → LIMIT 1000
       ↓
Ответ  ← [ ~100-500ms ] ИТОГО
```

## 🔍 Детальная логика WHERE и HAVING

```sql
WHERE 
    -- Фильтр 1: Только студенты
    p.person_type = 'student'
    
    -- Фильтр 2: Исключаем Россию
    AND pc.country_code != 'RU' 
    AND pc.country_name != 'РОССИЯ'
    
    -- Фильтр 3: Опциональный фильтр по стране
    AND (
        p_country = 'all'                    -- Все страны
        OR pc.country_name = p_country      -- Конкретная страна
    )

HAVING 
    -- Фильтр 4: Количество дней отсутствия
    days_missing >= p_days_threshold
```

## 💡 Ключевые особенности

### ✅ Что делает:
- ✅ Находит иностранных студентов
- ✅ Исключает российских студентов
- ✅ Рассчитывает дни отсутствия
- ✅ Фильтрует по порогу дней
- ✅ Показывает последнее место визита
- ✅ Сортирует по критичности

### ⚠️ Ограничения:
- ⚠️ Максимум 1000 записей
- ⚠️ Требует наличие events (проходов)
- ⚠️ NULL для студентов без upn/card_number
- ⚠️ Зависит от структуры БД СКУД

### 🚀 Производительность:
- 🚀 Индексы критичны для скорости
- 🚀 ~100-500ms на 10K студентов
- 🚀 Можно кешировать результат
