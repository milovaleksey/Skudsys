# Отладка проблемы "Донат считает 2109 вместо 2360"

## Проблема
В массиве данных 65 записей, сумма = 2360, но диаграмма показывает 2109.

## Причина
В исходных данных была запись:
```json
{"country": null, "students_count": 1}
```

Эта запись и другие некорректные фильтруются компонентом.

## Решение

### 1. Добавлена отладка в компонент
`/components/DonutChartCard.tsx` теперь выводит в консоль:
- Исходные данные
- Количество записей
- Отфильтрованные данные
- Пропущенные записи
- Пошаговый подсчет суммы
- Итоговую сумму

### 2. Добавлена отладка в хук
`/hooks/useForeignStudentsMQTT.ts` выводит:
- Данные, полученные с сервера
- Количество стран
- Общую сумму из данных
- Некорректные записи (null, пустые)

### 3. Фильтрация данных
Компонент теперь фильтрует:
```typescript
.filter(item => {
  const hasCountry = item.country && item.country.trim() !== '';
  const hasValue = typeof item.students_count === 'number' && item.students_count > 0;
  return hasCountry && hasValue;
})
```

**Отфильтровываются:**
- `country === null` или пустая строка
- `students_count === 0` или не число

## Что было не так?

### Вариант 1: Запись с `country: null`
```json
{"country": null, "students_count": 1}
```
Эта запись теперь фильтруется.

### Вариант 2: Неполная загрузка данных
Возможно, не все данные доходили до компонента.
Теперь логи покажут точное количество.

### Вариант 3: Типы данных
Возможно, `students_count` приходил как строка `"123"`.
Теперь используется `Number(item.students_count)`.

## Рекомендации

### 1. Очистите данные в источнике
Если данные приходят из базы:
```sql
-- Найти записи без страны
SELECT * FROM students WHERE country IS NULL OR country = '';

-- Обновить или удалить
UPDATE students SET country = 'Без гражданства' WHERE country IS NULL;
```

### 2. Добавьте категорию "Без гражданства"
Вместо фильтрации, можно показать:
```json
{"country": "Лицо без гражданства", "students_count": 9}
```
(У вас уже есть такая запись!)

### 3. Используйте логи для мониторинга

- Сколько записей пришло
