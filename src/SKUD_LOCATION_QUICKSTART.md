# 🚀 Быстрый старт: Страница "Где находится человек"

## ⚡ За 3 шага:

### 1️⃣ Создайте процедуры в MySQL

Откройте файл `/SKUD_LOCATION_PROCEDURES_TEMPLATE.sql` и создайте процедуры в вашей базе данных СКУД:

```sql
-- Процедура 1: Поиск по ФИО
CALL sp_get_last_entry_event('Милов', 'Алексей', 'Сергеевич');

-- Процедура 2: Поиск по UPN
CALL sp_get_last_entry_by_upn('a.s.milov@utmn.ru');
```

**⚠️ ВАЖНО:** Адаптируйте процедуры под вашу структуру базы данных!

---

### 2️⃣ Перезапустите backend

```bash
cd backend
npm start
```

---

### 3️⃣ Тестируйте

Откройте страницу **"Где находится человек"** и введите:
- **По ФИО:** `Милов Алексей Сергеевич`
- **По UPN:** `a.s.milov@utmn.ru`

---

## 📋 Что возвращает процедура:

```sql
+------------------+------------------------------------------+
| Поле             | Пример                                   |
+------------------+------------------------------------------+
| full_name        | "Милов Алексей Сергеевич"                |
| upn              | "a.s.milov@utmn.ru"                      |
| card_number      | "1446738"                                |
| department       | "Институт математики и компьютерных наук"|
| person_type      | "employee" или "student"                 |
| checkpoint_name  | "Главный вход, корпус А"                 |
| location         | "1 этаж, турникет №3"                    |
| event_time       | "2026-03-03 14:35:22"                    |
+------------------+------------------------------------------+
```

## ❓ Проблемы?

### Процедура не найдена:
```json
{
  "message": "Хранимая процедура sp_get_last_entry_event не найдена"
}
```
**Решение:** Создайте процедуры из шаблона

### Человек не найден:
```json
{
  "message": "Человек не найден или нет записей о проходах"
}
```
**Решение:** Проверьте данные в базе или используйте другие тестовые данные
