## 🎯 Жизненный цикл события

```
┌────────────────────────────────────────────────────────────┐
│ 1. MQTT событие приходит                                   │
└──────────────────────────┬─────────────────────────────────┘
                           ▼
┌────────────────────────────────────────────────────────────┐
│ 2. MQTT Service получает и сохраняет                       │
│                                                            │
│    handleMessage(topic, message) {                         │
│      const badEvents = JSON.parse(message);                │
│      engineeringController.addBadEvents(badEvents);        │
│      this.emit('bad-event', badEvents);                    │
│    }                                                       │
└──────────────────────────┬─────────────────────────────────┘
                           ▼
┌────────────────────────────────────────────────────────────┐
│ 3. Engineering Controller сохраняет в памяти               │
│                                                            │
│    badEvents = [...newEvents, ...badEvents].slice(0,10000) │
└──────────────────────────┬─────────────────────────────────┘
                           ▼
┌────────────────────────────────────────────────────────────┐
│ 4. WebSocket Server отправляет ВСЕМ клиентам               │
│                                                            │
│    clients.forEach(client => {                             │
│      client.send({ type: 'mqtt-message', data: events })   │
│    })                                                      │
└──────────────────────────┬─────────────────────────────────┘
                           ▼
┌──────────────────────────────────────────────────────��─────┐
│ 5. Frontend обновляет таблицу                              │
│                                                            │
│    setBadEvents(prev => [...newEvents, ...prev])          │
└────────────────────────────────────────────────────────────┘
```

## 📈 Масштабирование

### Текущая реализация (in-memory):
```
┌──────────┐
│ Backend  │  Хранит 10000 событий
│ Process  │  ~1-5 MB RAM
└────┬─────┘
     │
     ├─ Client 1  (получает 1000)
     ├─ Client 2  (получает 1000)
     ├─ Client 3  (получает 1000)
     └─ Client N  (получает 1000)

Проблема: При перезапуске теряется история
```

### Production решение (с БД):
```
┌──────────┐     ┌──────────┐
│ Backend  │────►│  MySQL   │
│ Process  │◄────│  Database│
└────┬─────┘     └──────────┘
     │           Хранит ВСЕ события
     │           Неограниченно
     │
     ├─ Client 1  (получает последние 1000)
     ├─ Client 2  (получает последние 1000)
     ├─ Client 3  (получает последние 1000)
     └─ Client N  (получает последние 1000)

✅ При перезапуске история сохраняется
✅ Можно делать SQL запросы и аналитику
✅ Архив событий за любой период
```
