// Динамическое извлечение из MQTT топика
const buildings = useMemo(() => {
  const uniqueBuildings = new Set<string>();
  
  // Извлекаем из топ локаций
  if (topLocations && topLocations.length > 0) {

      if (location.name) {
        // "Корпус №5 - Главный вход" → "Корпус №5"

## 🚀 Преимущества:

1. ✅ **Всегда актуальный список** - берется из реальных данных MQTT
2. ✅ **Автоматическое обновление** - при изменении конфигурации список обновляется
3. ✅ **Умная сортировка** - корпуса сортируются по номерам (№1, №2, №5, №9)
4. ✅ **Парсинг названий** - извлекает "Корпус №5" из "Корпус №5 - Главный вход"
5. ✅ **Уникальные значения** - каждое здание встречается только один раз

## 📊 Как работает:

### 1. MQTT присылает топ зоны:
```json
{
  "top_zones": [
    { "name": "Корпус №5 - Главный вход", "count": 1500 },
    { "name": "Корпус №5 - Боковой вход", "count": 800 },
    { "name": "Корпус №3 - Вход А", "count": 1200 },
    { "name": "Общежитие №1 - Ресепшен", "count": 900 },
    { "name": "Библиотека - Читальный зал", "count": 600 }
  ]
}
```

### 2. Алгоритм извлекает уникальные здания:
- "Корпус №5 - Главный вход" → `Корпус №5`
- "Корпус №5 - Боковой вход" → `Корпус №5` (дубликат, пропускаем)
- "Корпус №3 - Вход А" → `Корпус №3`
- "Общежитие №1 - Ресепшен" → `Общежитие №1`
- "Библиотека - Читальный зал" → `Библиотека`

### 3. Результат в фильтре:
```
Все корпуса
Корпус №3
Корпус №5
Общежитие №1
Библиотека
```

## 🎨 Умная сортировка:

### Алгоритм:
```javascript
sortedBuildings.sort((a, b) => {
  // Извлекаем номера
  const numA = a.match(/№(\d+)/)?.[1];
  const numB = b.match(/№(\d+)/)?.[1];
  
  // Если оба имеют номера - сортируем численно
  if (numA && numB) {
    return parseInt(numA) - parseInt(numB);
  }
  
  // Иначе - алфавитный порядок
  return a.localeCompare(b, 'ru');
});
```

### Пример сортировки:
```
✅ Корректно:
   Корпус №1
   Корпус №2
   Корпус №5
   Корпус №9
   Корпус №10
   Библиотека
   Общежитие №1

❌ Неправильно (алфавитная):
   Корпус №1
   Корпус №10
   Корпус №2
   Корпус №5
```
