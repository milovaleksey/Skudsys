# 🔧 Инженерный раздел - Документация

## Обзор

**Инженерный раздел** - это специализированная страница системы безопасности ТюмГУ для мониторинга аномальных событий СКУД и управления правилами доступа.

## 🎯 Основной функционал

### 1. Мониторинг аномальных событий
- **Live-обновления** через MQTT топик `Skud/baddialsevent`
- **Таблица событий** с полной информацией:
  - Дата и время события
  - Тип события (например: "Идентификатор не зарегистрирован")
  - ФИО пользователя
  - UPN (email)
  - Зона и подзона
  - Устройство
  - ID карты

### 2. Фильтрация данных
- **По датам**: выбор периода (от/до)
- **По типу события**: фильтр по различным типам аномалий
- **По устройству**: выбор конкретного турникета/считывателя
- **Поиск**: по ФИО, устройству, ID карты

### 3. Экспорт данных
- Выгрузка в **Excel** с форматированием
- Все отфильтрованные данные
- Имя файла содержит период: `Аномальные_события_ДД.ММ.ГГГГ_ДД.ММ.ГГГГ.xlsx`

### 4. Управление правилами доступа
- **Создание правил**: отдел + шаблон доступа + тип пользователя
- **Редактирование** существующих правил
- **Удаление** правил с подтверждением
- **Активация/деактивация** правил одним кликом
- **Типы применения**:
  - Сотрудники
  - Студенты
  - Все (сотрудники и студенты)

```typescript
interface BadEvent {
  time_label: string;           // ISO datetime
  Тип_события: string;          // Тип аномалии
  ФИО_пользователя: string;     // Полное имя
  UPN: string | null;           // Email
  Zone: string | null;          // Зона
  Child_Zone: string | null;    // Подзона
```

## 🚀 Использование

### 1. Просмотр аномальных событий

1. Перейдите в "Инженерный раздел" в боковом меню
2. События отображаются автоматически (live)
3. Используйте фильтры для уточнения
4. Нажмите "Экспорт" для выгрузки в Excel

### 2. Создание правила доступа

1. Нажмите "Создать правило"
2. Заполните форму:
   - **Отдел**: название подразделения
   - **Шаблон доступа**: описание прав (например: "Полный доступ 24/7")
   - **Применяется к**: выберите тип пользователя
3. Нажмите "Сохранить"

### 3. Редактирование правила

1. Найдите правило в таблице
2. Нажмите кнопку редактирования (✏️)
3. Измените данные
4. Сохраните

### 4. Активация/деактивация

- Кликните по статусу правила ("Активно"/"Неактивно")
- Правило изменит состояние мгновенно

## 🐛 Решение проблем

### MQTT не подключается
1. Проверьте `.env`: `MQTT_ENABLED=true`
2. Проверьте настройки брокера: `MQTT_BROKER`, `MQTT_PORT`
3. Проверьте логи backend: `[MQTT]` префикс

### События не отображаются
1. Убедитесь что MQTT подключен (зеленый индикатор)
2. Проверьте публикацию в топик `Skud/baddialsevent`
3. Проверьте формат JSON (должен быть массив)

### Нет доступа к странице
1. Проверьте роль пользователя (`admin` или `security`)
2. Выполните миграцию: `node backend/add-engineering-permission.js`
3. Перезапустите backend сервер
