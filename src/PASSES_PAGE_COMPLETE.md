### ✅ Database (MySQL)

**Файл:** `/SKUD_PASSES_PROCEDURES_TEMPLATE.sql`

**Процедуры:**
1. `sp_get_passes_by_fio(lastName, firstName, middleName, dateFrom, dateTo)`
2. `sp_get_passes_by_upn(upn, dateFrom, dateTo)`

**Возвращаемые поля:**
- `id` - ID записи
- `event_time` - Время прохода
- `full_name` - ФИО
- `upn` - Email
- `card_number` - Номер карты
- `checkpoint_name` - Точка прохода
- `event_name` - **Название события** (новое поле!)
- `direction` - Направление (in/out)
- `building` - Здание

## 📋 Инструкция по установке:

### 1. Создайте процедуры в MySQL

```bash
mysql -u root -p skud_database < /SKUD_PASSES_PROCEDURES_TEMPLATE.sql
```

Или вручную скопируйте и выполните процедуры из файла.

**⚠️ ВАЖНО:** Адаптируйте процедуры под вашу структуру БД!

### 2. Проверьте процедуры

```sql
-- Проверка существования
SHOW PROCEDURE STATUS WHERE Db = 'skud_database';

-- Тестовый вызов по ФИО
CALL sp_get_passes_by_fio(
    'Милов', 
    'Алексей', 
    'Сергеевич',
    '2026-03-03 00:00:00',
    '2026-03-03 23:59:59'
);

-- Тестовый вызов по UPN
CALL sp_get_passes_by_upn(
    'a.s.milov@utmn.ru',
    '2026-03-03 00:00:00',
    '2026-03-03 23:59:59'
);
```

### 3. Перезапустите backend

```bash
cd backend
npm start
```

### 4. Протестируйте через frontend

1. Откройте страницу "Отчет о проходах"
2. Обратите внимание - **диапазон дат уже установлен на сегодняшний день**
3. Введите ФИО или UPN (автоопределение типа поиска)
4. Нажмите "Найти"
5. Попробуйте экспортировать в Excel

## ✨ Особенности реализации:

### 🎯 Автоопределение типа поиска

При вводе текста в поле поиска:
- Если ввод содержит `@` → **автоматически выбирается "По UPN"**
- Если ввод НЕ содержит `@` → **автоматически выбирается "По ФИО"**

Пример:
- Ввели `petrova@study.utmn.ru` → чекбокс автоматически переключился на UPN ✅
- Ввели `Петрова Мария` → чекбокс автоматически переключился на ФИО ✅

### 📅 Дефолтные даты - сегодня

При открытии страницы:
- **Дата от:** сегодня 00:00:00
- **Дата до:** сегодня 23:59:59

Это позволяет сразу искать проходы за текущий день без ручной установки дат!

### 📊 Экспорт в Excel

- Кнопка "Excel" **неактивна**, если нет данных
- При экспорте создается файл `passes_report_YYYY-MM-DD.xlsx`
- Экспортируются все найденные записи с красивым форматированием
- Столбцы: **Время, ФИО, UPN, Номер карты, Событие, Точка прохода, Направление, Здание**
