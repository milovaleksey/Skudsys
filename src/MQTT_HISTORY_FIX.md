# ✅ Исправлено: История MQTT событий для новых клиентов

## 🎯 Проблема

При подключении нового клиента к странице **"Инженерный раздел"** он **НЕ получал историю** ранее полученных аномальных событий из MQTT топика `Skud/baddialsevent`.

### Симптомы:
- ✅ Новые события приходят в реальном времени
- ❌ Старые события (полученные до подключения) не отображаются
- ❌ При обновлении страницы таблица становится пустой

---

## 🔍 Причина

**WebSocket сервер** (`/backend/src/websocket/mqtt.ws.js`) при подключении нового клиента отправлял:
- ✅ Карточки MQTT
- ✅ Аналитику
- ✅ Статус подключения

**НО НЕ отправлял историю аномальных событий!** ❌

---

## ✅ Решение

### 1. **Добавлена отправка истории при подключении**

**Файл:** `/backend/src/websocket/mqtt.ws.js`

```javascript
// Получаем контроллер для доступа к истории событий
const engineeringController = require('../controllers/engineering.controller');

// Отправляем текущие данные при подключении
const initialData = {
  type: 'initial',
  cards: mqttService.getCardsWithValues(),
  analytics: mqttService.getAnalyticsData(),
  status: mqttService.getStatus(),
};
ws.send(JSON.stringify(initialData));

// ✨ НОВОЕ: Отправляем историю аномальных событий при подключении
const badEventsHistory = engineeringController.getBadEventsHistory();
if (badEventsHistory && badEventsHistory.length > 0) {
  console.log(`[WebSocket] 📜 Отправка истории: ${badEventsHistory.length} аномальных событий новому клиенту`);
  ws.send(JSON.stringify({
    type: 'mqtt-message',
    topic: 'Skud/baddialsevent',
    data: badEventsHistory,
  }));
}
```

### 2. **Добавлен метод для получения истории**

**Файл:** `/backend/src/controllers/engineering.controller.js`

```javascript
/**
 * Получить историю аномальных событий
 * Используется WebSocket сервером для отправки истории новому клиенту
 */
exports.getBadEventsHistory = () => {
  // Возвращаем последние 1000 событий
  return badEvents.slice(0, 1000);
};
```

## 📊 Хранение данных

### Backend (in-memory):
```javascript
// В engineering.controller.js
let badEvents = []; // Массив событий

exports.addBadEvent = (event) => {

  }
```

## ⚠️ Важные замечания

### 1. **In-memory хранилище**
События хранятся в памяти Node.js процесса:
- ✅ **Плюс:** Быстрый доступ
- ❌ **Минус:** При перезапуске backend история теряется

### 2. **Для production**
Рекомендуется сохранять события в БД:

```javascript
// Пример с MySQL
exports.addBadEvent = async (event) => {
  // Сохраняем в БД
  await db.query(
    'INSERT INTO bad_events (time_label, event_type, fio, device) VALUES (?, ?, ?, ?)',
    [event.time_label, event.Тип_события, event.ФИО_пользователя, event.Device]
  );
  
  // Также держим в памяти для быстрого доступа
  badEvents.unshift(event);
  if (badEvents.length > 10000) {
    badEvents = badEvents.slice(0, 10000);
  }
};

exports.getBadEventsHistory = async () => {
  // Загружаем из БД
  const result = await db.query(
    'SELECT * FROM bad_events ORDER BY time_label DESC LIMIT 1000'
  );
  return result;
};
```

### 3. **Производительность**
При большом количестве клиентов:
- Каждому отправляется 1000 событий при подключении
- Можно добавить пагинацию или ленивую загрузку
