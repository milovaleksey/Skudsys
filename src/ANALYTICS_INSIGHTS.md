# 📊 Аналитические выводы из датасета СКУД ТюмГУ

## 📁 Структура данных

**Формат:** JSON массив с агрегированными событиями
**Поля:**
- `event_date` - дата события (ISO 8601)
- `root_zone_name` - название корневой зоны
- `total_events` - количество событий за день в зоне

**Период данных:** 17 июня 2025 - середина июля 2025 (~1 месяц)

---

## 🎯 10 типов аналитики, которые можно получить

### 1️⃣ **Общая статистика (Statistics)**

**Что показывает:**
- Всего проходов за период
- Количество уникальных зон
- Среднее количество проходов в день
- Пиковый день активности

**Зачем это нужно:**
- Быстрая оценка загруженности кампуса
- KPI для безопасности и планирования
- Сравнение периодов

**Пример:**
```
Всего проходов: 250,000
Уникальных зон: 17
Среднее в день: 8,333
Пик: 26 июня 2025 - 14,679 проходов
```

---

### 2️⃣ **Динамика по дням (Time Series)**

**Что показывает:**
- График активности по дням
- Тренды (рост/спад)
- Аномалии и выбросы

**Ключевые наблюдения из данных:**
- **Будние дни:** 7,000-9,000 проходов
- **Выходные (суббота, воскресенье):** резкий спад до 1,500-3,000
- **Пики:** середина недели (среда-четверг)
- **Провалы:** 28 июня (воскресенье) - всего 2,926 проходов

**Зачем это нужно:**
- Прогнозирование нагрузки
- Планирование охраны и уборки
- Выявление необычных ситуаций

### 4️⃣ **Распределение по дням недели (Weekday Pattern)**

**Наблюдения:**

```
Понедельник (Пн): 7,500 проходов (средне)
Вторник (Вт):     8,500 проходов (↑ высоко)
Среда (Ср):       9,000 проходов (↑↑ пик)
Четверг (Чт):     8,700 проходов (↑ высоко)
Пятница (Пт):     7,200 проходов (средне)
Суббота (Сб):     2,000 проходов (↓↓ низко)
Воскресенье (Вс): 1,500 проходов (↓↓↓ минимум)
```

**Выводы:**
- Середина недели - пик учебной активности
- Пятница - начало спада (студенты разъезжаются)
- Выходные - только общежития активны

**Зачем это нужно:**
- График работы персонала
- Планирование мероприятий
- Энергосбережение (отключение в выходные)

### 5️⃣ **Сравнение топ-5 зон по дням (Multi-Series)**

**Что показывает:**
- Как меняется активность в разных корпусах
- Какие корпуса работают синхронно
- Какие зоны имеют свой график

**Наблюдения:**
- **Корпус №5 и №4** - похожие паттерны (рядом расположены?)
- **Общежитие 5** - высокая активность в выходные (студенты остаются)
- **Корпус № 4 Приемная комиссия** - всплеск с 23-25 июня (приемная кампания!)
- **Корпус №17** - низкая активность (возможно, специализированный)

**Зачем это нужно:**
- Понимание взаимосвязей между зонами
- Оптимизация маршрутов студентов
- Планирование расписания

### 6️⃣ **Категоризация зон (Pie Chart)**

**Распределение по типам:**

```
🏫 Учебные корпуса:        60-65%
🏠 Общежития:              30-35%
📝 Приемная комиссия:      2-3%
📚 Библиотеки/Прочие:      2-3%
```

**Выводы:**
- Основная активность - учебный процесс
- Общежития - значительная доля (много иногородних)
- Приемная комиссия - сезонная активность

**Зачем это нужно:**
- Стратегическое планирование развития
- Бюджетирование систем безопасности
- Анализ структуры контингента

## 🚀 Дополнительная аналитика (Advanced)

### 1. **Корреляционный анализ**
- Связь активности между корпусами
- Влияние погоды на посещаемость (если добавить метео-данные)

### 2. **Прогнозирование (Machine Learning)**
- Предсказание активности на следующую неделю
- Аномальное поведение (security alerts)

### 3. **Пространственная аналитика**
- Тепловые карты на плане кампуса
- Маршруты перемещений между зонами

### 4. **Анализ пиковых нагрузок**
- Часы пик в учебных корпусах
- Очереди в столовых (если есть данные по времени)

### 5. **Энергоэффективность**
- Корреляция с потреблением электроэнергии
- Оптимизация отопления/вентиляции

## 📋 Рекомендации по использованию

### Для руководства университета:
1. **Мониторинг общей активности** - контроль заполняемости кампуса
2. **Оптимизация расписания** - перенос занятий в менее загруженные дни
3. **Планирование ремонтов** - в периоды низкой активности

### Для службы безопасности:
1. **Распределение охраны** - больше в топ-зоны, меньше в выходные
2. **Аномалии** - внезапный всплеск/спад активности
3. **Контроль доступа** - анализ попыток незаконного проникновения

### Для административного отдела:
1. **График работы персонала** - уборка, техподдержка
2. **Коммунальные услуги** - отключение в неактивные периоды
3. **Планирование мероприятий** - выбор оптимального времени

### Для студенческого самоуправления:
1. **Мероприятия** - когда максимальная явка
2. **Загруженность библиотек** - планирование посещений
3. **Работа общежитий** - контроль комфорта проживания

## 🔧 Техническая реализация

### MQTT топик для конфигурации:
```
Skud/analytics/config
```

### MQTT топик для данных:
```
Skud/analytics/events/aggregated
```

### Формат публикации конфига:
```bash
mosquitto_pub -h localhost -p 1883 \
  -t "Skud/analytics/config" \
  -f ANALYTICS_CONFIG_EXAMPLE.json \
  -r  # retained message
```

### Формат публикации данных:
```bash
mosquitto_pub -h localhost -p 1883 \
  -t "Skud/analytics/events/aggregated" \
  -f /imports/event-data-1.json
```

---

**Итого: Система позволяет получить 10+ типов аналитики из одного датасета, полностью конфигурируемых через MQTT!** 🎯
