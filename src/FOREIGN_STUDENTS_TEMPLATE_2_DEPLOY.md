# 🎯 Развертывание шаблона "Пропавшие студенты"

## 📋 Обзор

Второй шаблон отчетов "Пропавшие студенты" позволяет находить иностранных студентов, которые отсутствуют в системе СКУД более N дней.

### Backend:
- ✅ Контроллер `foreignStudentsController.getMissingStudents()`
- ✅ Роут `GET /api/foreign-students/missing`
- ✅ Формат ответа приведен к `PassRecord`

### Frontend:
- ✅ API метод `skudApi.getForeignStudentsMissing()`
- ✅ UI форма с выбором страны и порога дней
- ✅ Таблица результатов с колонками:
  - ФИО
  - Логин/Почта
  - Страна (синий бейджик)
  - Последний визит
  - Место
  - Дней отсутствия (красный ≥7 дней, желтый <7 дней)
- ✅ Экспорт в Excel

## 📊 Структура данных

### Входные параметры:
- `p_country` (VARCHAR): 'all' или название страны (например, 'КАЗАХСТАН')
- `p_days_threshold` (INT): минимальное количество дней отсутствия

### Возвращаемые поля:
- `id` - ID записи
- `fio` - Полное имя студента
- `upn` - Логин/почта
- `card_number` - Номер карты доступа
- `country` - Название страны
- `last_seen` - Дата/время последнего визита
- `last_location` - Место последнего визита
- `days_missing` - Количество дней отсутствия

## 🎨 UI компонент

### Форма поиска:
- **Выбор страны**: Dropdown с динамическим списком стран из MQTT
  - "Все страны (кроме России)" - по умолчанию
  - Список стран загружается из `countryStats` (без России)
  
- **Порог дней отсутствия**: Number input
  - По умолчанию: 3 дня
  - Минимум: 1 день

### Таблица результатов:
- Цветовая индикация дней отсутствия:
  - 🔴 **Красный** (≥7 дней): критическое отсутствие
  - 🟡 **Желтый** (<7 дней): предупреждение

## 🔍 Логика работы процедуры

1. **Фильтрация по типу**:
   - Только `person_type = 'student'`
   - Только НЕ Россия (`country_code != 'RU'`)

2. **Группировка**:
   - По каждому студенту
   - Находим MAX(event_time) - последний визит

3. **Расчет отсутствия**:
   - `DATEDIFF(NOW(), MAX(event_time))` - количество дней

4. **Фильтрация**:
   - `days_missing >= p_days_threshold`
   - Опционально по стране

5. **Сортировка**:
   - По убыванию дней отсутствия
   - Затем по ФИО (алфавит)

## ⚠️ Важные замечания

### Структура базы данных:
Хранимая процедура предполагает следующие таблицы в БД СКУД:
- `persons` - таблица персон
- `person_countries` - справочник стран
- `events` - события проходов
- `access_points` - точки доступа

- `persons`: id, last_name, first_name, middle_name, upn, card_number, country_id, person_type
- `person_countries`: id, country_code, country_name
- `events`: person_id, event_time, access_point_id
- `access_points`: id, name
