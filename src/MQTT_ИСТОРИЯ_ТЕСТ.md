# 🧪 Тестирование истории MQTT событий

## 🎯 Что тестируем

Проверяем что новый клиент получает **историю** аномальных событий из топика `Skud/baddialsevent` при подключении к странице "Инженерный раздел".

---

## ✅ Предварительные требования

1. **Backend запущен:**
   ```bash
   cd backend
   node src/server.js
   ```

2. **MQTT брокер доступен** (настроен в `.env`)

3. **Frontend запущен:**
   ```bash
   npm run dev
   # или
   cd frontend && npm run dev
   ```

4. **Вы залогинены** в систему

---

## 🧪 Тест 1: Первое подключение

### Цель
Проверить что при первом открытии страницы клиент получает историю.

### Шаги

1. **Откройте консоль браузера** (F12)

2. **Перейдите на страницу "Инженерный раздел"**
   ```
   http://localhost:5173/engineering
   ```

3. **Проверьте консоль браузера:**
   
   Должны увидеть:
   ```javascript
   [Engineering] Подключение к WebSocket: ws://localhost:3000/ws/mqtt?token=...
   ✅ [Engineering] WebSocket подключен
   [Engineering] Получено WebSocket сообщение: {
     type: "mqtt-message",
     topic: "Skud/baddialsevent",
     data: [...]  // ← Массив событий (история)
   }
   ```

4. **Проверьте таблицу:**
   - ✅ Таблица "Аномальные проходы" заполнена данными
   - ✅ Видны события с временными метками
   - ✅ Индикатор MQTT показывает "Подключено" (зеленый)

### Ожидаемый результат
✅ **УСПЕХ:** Таблица содержит события из истории

❌ **ОШИБКА:** Таблица пустая → проверьте backend логи

## 🔍 Проверка backend логов

### Логи при подключении клиента

```bash
[MQTT WebSocket] Новое подключение
[WebSocket] ✅ Аутентифицирован: admin
[WebSocket] 📜 Отправка истории: 1000 аномальных событий новому клиенту
```

Если НЕТ строки "📜 Отправка истории":
- ❌ История пустая (badEvents.length = 0)
- ❌ Метод getBadEventsHistory() не вызывается

### Логи при получении MQTT события

```bash
[MQTT] 🚨 Получено 1 аномальных событий из топика Skud/baddialsevent
[MQTT] 🚨 Испускаем событие 'bad-event' для WebSocket клиентов
[WebSocket] 🚨 Рассылка аномальных событий СКУД клиентам
[WebSocket] 🚨 Подключено клиентов: 1
[WebSocket] 🚨 Данные события: [...]
[WebSocket] ✅ Событие отправлено 1 клиентам
```

## ⚠️ Troubleshooting

### Проблема: Таблица пустая при загрузке

**Возможные причины:**

1. **Backend не хранит события**
   ```bash
   # Проверьте в backend консо��и:
   grep "badEventsCount" logs
   ```
   
   Решение:
   - Опубликуйте тестовое событие в MQTT
   - Перезапустите backend

2. **WebSocket не отправляет историю**
   ```bash
   # Должна быть строка:
   [WebSocket] 📜 Отправка истории: ...
   ```
   
   Решение:
   - Проверьте что изменения в `/backend/src/websocket/mqtt.ws.js` применены
   - Перезапустите backend

3. **Frontend не обрабатывает сообщения**
   
   Решение:
   - Проверьте консоль браузера на ошибки
   - Убедитесь что WebSocket подключен

### Проблема: История теряется при перезапуске

**Это нормально!** События хранятся в памяти (in-memory).

Решение для production:
- Сохраняйте события в MySQL
- См. `/MQTT_HISTORY_FIX.md` раздел "Для production"

### Проблема: Новые события не приходят

**Проверьте:**

1. MQTT брокер работает:
   ```bash
   mosquitto_sub -h localhost -p 1883 -t "Skud/#" -v
   ```

2. Backend подписан на топик:
   ```bash
   # В логах должно быть:
   [MQTT] ✅ Подписан на топик: Skud/baddialsevent
   ```

3. WebSocket подключен:
   ```bash
   # В консоли браузера:
   ✅ [Engineering] WebSocket подключен
   ```
