# 📊 Отчет по преподавателям

## 🎯 Назначение

Страница визуализации активности преподавателей с интеграцией данных о проходах через СКУД и расписанием занятий. Позволяет отслеживать местоположение, соблюдение расписания и выявлять нарушения (опоздания/ранние уходы).

- **`/components/MainPage.tsx`** — добавлен роут для страницы

## 🎨 Основной функционал

### 1️⃣ Выбор преподавателя
- Dropdown-список преподавателей с указанием отдела
- Mock данные: 3 преподавателя из разных институтов

### 2️⃣ Выбор периода

#### Цветовая кодировка проходов
| Цвет | Тип местоположения | Описание |
|------|-------------------|----------|
| 🟢 **Зеленый** | `building` | Вход в корпус |
| 🟠 **Оранжевый** | `floor` | Этаж внутри корпуса |
| 🔴 **Красно-оранжевый** | `room` | Аудитория |
| ⚪ **Серый** | `uncontrolled` | Вне территории |

## 📖 Расписание занятий

### Визуальное отображение
- **Полупрозрачные блоки** накладываются поверх трека проходов
- **Пунктирная рамка** для границ занятия
- **Цвета по типам:**
  - 🟣 **Фиолетовый** — Лекции
  - 🔵 **Синий** — Практики
  - 🟠 **Оранжевый** — Лабораторные работы

### Tooltip при наведении
При наведении мыши на блок расписания отображается:
```
📚 Методика переработки ядерного топлива
🕐 08:00 - 09:30
🏫 Аудитория: К1-301
⚠️ Опоздание на 5 мин (если есть)
```

## ⚠️ Определение нарушений

### Логика проверки

#### 🔴 Опоздание
- Преподаватель пришел **> 3 минуты после начала** занятия
- Пример: Лекция в 08:00, пришел в 08:05 → опоздание на 5 мин

#### 🟠 Ранний уход
- Преподаватель ушел **> 5 минут до окончания** занятия
- Пример: Лекция до 09:30, ушел в 09:20 → ушел на 10 мин раньше

### Визуализация нарушений
- Красный значок ⚠️ в правом верхнем углу блока расписания
- Красная/оранжевая пунктирная рамка
- Информация в tooltip

## 📦 Структура данных

### PassEvent (проходы)
```typescript
interface PassEvent {
  time: string;           // ISO формат: "2026-03-16T06:12:00Z"
  location: string;       // Название точки прохода
  locationType: 'building' | 'floor' | 'room' | 'uncontrolled';
  building?: string;      // Корпус (К1, К2, ...)
  floor?: string;         // Этаж (3, Т, ...)
  room?: string;          // Аудитория (301, Т351, ...)
}
```

**Пример:**
```json
{
  "time": "2026-03-16T07:59:00Z",
  "location": "К1 Этаж 3 301",
  "locationType": "room",
  "building": "К1",
  "floor": "3",
  "room": "301"
}
```

### ScheduleEvent (расписание)
```typescript
interface ScheduleEvent {
  startTime: string;      // HH:MM формат: "08:00"
  endTime: string;        // HH:MM формат: "09:30"
  title: string;          // Название занятия
  room: string;           // Аудитория: "К1-301"
  type: 'lecture' | 'practice' | 'lab';
}
```

**Пример:**
```json
{
  "startTime": "08:00",
  "endTime": "09:30",
  "title": "Методика переработки ядерного топлива",
  "room": "К1-301",
  "type": "lecture"
}
```

### DaySchedule (день)
```typescript
interface DaySchedule {
  date: string;           // YYYY-MM-DD: "2026-03-16"
  passes: PassEvent[];    // Массив проходов
  schedule: ScheduleEvent[]; // Массив занятий
}
```

## 🔧 Mock-генераторы

### `generateMockTeachers()`
Возвращает массив из 3 преподавателей:
```typescript
[
  { id: '1', name: 'Иванов Иван Иванович', department: 'Институт математики и компьютерных наук' },
  { id: '2', name: 'Петрова Мария Сергеевна', department: 'Институт физики и технологий' },
  { id: '3', name: 'Сидоров Алексей Петрович', department: 'Институт химии' }
]
```

### `generateMockSchedule(date: string)`
Генерирует расписание в зависимости от дня недели:
- **Пн, Ср, Пт:** 3 занятия (лекции + лабораторная)
- **Вт, Чт:** 2 занятия (практика + консультация)
- **Сб, Вс:** Нет занятий

### `generateMockPasses(date: string)`
Генерирует реалистичные проходы:
- Приход в университет ~6:00
- Вход в аудитории перед занятиями (с 30% шансом опоздания 3-15 мин)
- Перемещения между корпусами
- Обеденный перерыв (выход из корпуса)
- 20% шанс раннего ухода (5-10 мин до конца)

## 🎯 Использование

### 1. Добавление прав доступа
В `RolesManagementPage` назначьте роли право **"Отчет по преподавателям"** (`teachers`).

### 2. Доступ к странице
После авторизации страница появится в меню (если есть права).

### 3. Работа с отчетом
1. Выберите преподавателя из списка
2. Выберите период (Неделя / Месяц)
3. Используйте навигацию для просмотра других периодов
4. Наведите мышь на элементы для деталей
5. Экспортируйте в Excel при необходимости

### `DayTimeline`
Компонент визуализации одного дня.

**Props:**
```typescript
interface DayTimelineProps {
  dayData: DaySchedule;
}
```

**Основные методы:**

- `hoveredEvent` — данные для tooltip (позиция + контент)
