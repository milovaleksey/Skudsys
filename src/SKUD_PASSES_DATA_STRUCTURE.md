## 📝 SQL SELECT (пример для процедуры sp_get_passes_by_fio)

```sql
SELECT 
    -- ОБЯЗАТЕЛЬНЫЕ ПОЛЯ
    al.id AS id,                                                          -- ID записи
    al.event_time AS event_time,                                          -- Время прохода
    CONCAT_WS(' ', p.last_name, p.first_name, p.middle_name) AS full_name, -- ФИО
    
    -- ОСНОВНЫЕ ПОЛЯ (могут быть NULL)
    p.upn AS upn,                                                         -- Email
    c.card_number AS card_number,                                         -- Номер карты
    ap.name AS checkpoint_name,                                           -- Точка прохода
    al.event_name AS event_name,                                          -- Название события
    
    -- ДОПОЛНИТЕЛЬНЫЕ ПОЛЯ (опциональные)
    al.direction AS direction,                                            -- Направление
    ap.building AS building,                                              -- Здание
    ap.location AS location                                               -- Местоположение
    
FROM access_logs al
LEFT JOIN persons p ON al.person_id = p.id
LEFT JOIN cards c ON p.card_id = c.id
LEFT JOIN access_points ap ON al.access_point_id = ap.id
WHERE 
    p.last_name = 'Милов'
    AND p.first_name = 'Алексей'
    AND p.middle_name = 'Сергеевич'
    AND al.event_time >= '2026-03-03 00:00:00'
    AND al.event_time <= '2026-03-03 23:59:59'
ORDER BY al.event_time DESC;
```

## 🎨 Форматы данных

### event_time (Время прохода)
- **Формат:** `YYYY-MM-DD HH:MM:SS`
- **Примеры:**
  - `2026-03-03 08:12:15`
  - `2026-03-03 14:35:22`
  - `2026-12-31 23:59:59`

### full_name (ФИО)
- **Формат:** `Фамилия Имя Отчество`
- **Конкатенация:** `CONCAT_WS(' ', p.last_name, p.first_name, p.middle_name)`
- **Примеры:**
  - `Милов Алексей Сергеевич`
  - `Петрова Мария Сергеевна`
  - `Иванов Иван` (без отчества)

### upn (Email)
- **Формат:** `username@domain.tld`
- **Примеры:**
  - `a.s.milov@utmn.ru` (сотрудник)
  - `petrova@study.utmn.ru` (студент)
  - NULL (если email не указан)

### card_number (Номер карты)
- **Формат:** Строка цифр
- **Примеры:**
  - `1446738`
  - `0987654321`
  - NULL (если карта не привязана)

### checkpoint_name (Точка прохода)
- **Формат:** Произвольный текст
- **Примеры:**
  - `Главный вход, корпус А`
  - `Библиотека`
  - `Турникет 1`
  - `Вход студенты, корпус Б`

### event_name (Название события) ⭐ НОВОЕ
- **Формат:** Произвольный текст
- **Примеры:**
  - `Вход разрешен`
  - `Выход`
  - `Доступ предоставлен`
  - `Проход по расписанию`
  - `Доступ запрещен`
  - `Проход через турникет`
  - NULL (если тип события не указан)

### direction (Направление)
- **Формат:** `in` или `out`
- **Примеры:**
  - `in` - вход
  - `out` - выход
  - NULL (если направление не определено)

### building (Здание)
- **Формат:** Произвольный текст
- **Примеры:**
  - `Корпус А`
  - `Главный корпус`
  - `Корпус Б`
  - NULL

### location (Местоположение)
- **Формат:** Произвольный текст
- **Примеры:**
  - `1 этаж, главный вход`
  - `3 этаж, читальный зал`
  - NULL

## ⚠️ ВАЖНО: Обработка NULL значений

Backend автоматически обрабатывает NULL значения:

```javascript
// В контроллере:
upn: result.upn || result.email || null,
cardNumber: result.card_number || result.cardNumber || null,
eventName: result.event_name || result.eventName || result.event_type || null,
```

Frontend отображает NULL как "—":

```jsx
<td>{record.eventName || '—'}</td>
<td>{record.upn || '—'}</td>
<td>{record.cardNumber || '—'}</td>
```
