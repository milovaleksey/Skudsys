# ✅ Исправление ошибки 400 при смене пароля - ЗАВЕРШЕНО

## 🎯 Проблема
Ошибка `"isActive" must be a boolean` при обновлении пользователя через API `/v1/users/:id`

## 💡 Причина
Фронтенд отправлял `isActive: 1` (число), а бэкенд ожидал `isActive: true/false` (boolean)

## ✅ Решение

### Backend изменения:
- ✅ Обновлены схемы валидации Joi для поддержки как boolean, так и числовых значений (0, 1)
- ✅ Добавлено преобразование числовых значений в boolean после валидации
- ✅ Добавлено поле `password` для смены пароля через API
- ✅ Добавлено логирование для отладки

### Frontend изменения:
- ✅ Создан компонент `PasswordRequirements.tsx` с визуальной валидацией пароля
- ✅ Обновлен `UsersSettingsPage.tsx` с интеграцией компонента требований к паролю
- ✅ Добавлены подсказки о минимальных требованиях (8 символов, заглавные/строчные буквы, цифры)

## 🔄 Что нужно сделать:

**ВАЖНО! Перезапустите backend-сервер:**

```bash
# Вариант 1: PM2
pm2 restart security-backend

# Вариант 2: systemd
sudo systemctl restart security-backend

# Вариант 3: Вручную
ps aux | grep node
kill -9 <PID>
cd backend && npm start
```

## ✨ Результат
После перезапуска:
- ✅ Ошибка 400 исчезнет
- ✅ Администратор сможет менять пароли пользователей
- ✅ Пользователи увидят визуальные подсказки при вводе пароля
- ✅ Все изменения будут записываться в audit log
